Testing Guide: WhatsApp Bot
Step 1: Meta Developer Console — Get credentials
Go to developers.facebook.com → Your App → WhatsApp → API Setup
You'll see:
Temporary access token (top of page) — expires every 24 hours, copy it
Phone Number ID — copy it (your test number's ID)
Scroll down to "To" field → enter your personal WhatsApp number → click Add number (you must do this to receive test messages)
Step 2: Update .env

WHATSAPP_ACCESS_TOKEN='paste_new_token_here'   ← changes every 24h
WHATSAPP_PHONE_NUMBER_ID='your_phone_number_id' ← stays the same
WHATSAPP_APP_SECRET='your_app_secret'           ← stays the same
                                                  (App Settings → Basic → App Secret)
Step 3: Set Webhook URL in Meta
WhatsApp → Configuration → Webhook Edit
URL: https://YOUR-NGROK-URL.ngrok-free.dev/webhook
Verify token: my_secret_verify_token (your WEBHOOK_VERIFY_TOKEN value)
Subscribe to: messages
Step 4: Start server correctly (every time)

# 1. Kill any old server first (run in PowerShell)
Stop-Process -Id (Get-NetTCPConnection -LocalPort 3000).OwningProcess -Force

# 2. Start ngrok (if not running)
ngrok http 3000

# 3. Start server (new terminal)
npm start
Important: If you restart the server, update the webhook URL in Meta IF ngrok URL changed. If you use ngrok's free static domain, it stays the same.

Step 5: Test checklist
 Ngrok is running: http://localhost:4040
 Server is running: http://localhost:3000/health returns {"status":"ok"}
 Webhook URL in Meta matches your current ngrok URL
 Access token in .env was copied within last 24 hours
 Your personal number is added as a test recipient in Meta console
 After any .env change, restart the server (kill old PID first)
