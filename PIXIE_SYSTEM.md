## 3. DOMAINS

Customer picks one of three paths:

### Option A — "I need a new domain"

- Pixie searches NameSilo for affordable options under $25/yr across 12 popular TLDs
- Shows top 5 (prefers .com, then cheapest)
- If none affordable, AI suggests similar names (e.g. "glowstudio" taken → "glowstudiohq", "glowsalonco" suggested)
- Customer picks one
- Combined Stripe link covers website + domain ($199 + actual domain price)
- After payment: domain registered automatically, DNS configured, Netlify alias added
- Free WHOIS privacy included
- **No auto-renewal** — customer chooses each year (renewal flow planned, see DOMAIN_RENEWAL_PLAN.md)

### Option B — "I already own a domain"

- Customer types their domain (e.g. mybiz.com)
- Pixie asks where they bought it (GoDaddy, Namecheap, Cloudflare, Hostinger, etc.)
- After payment: Pixie sends step-by-step DNS setup instructions tailored to that exact registrar
- AI generates registrar-specific instructions (works for any registrar including obscure ones)
- DNS records explained: A record + CNAME with exact values
- Cloudflare proxy warning included automatically when relevant

### Option C — "Skip for now"

- Site lives on the free Netlify preview URL
- Customer can come back anytime and add a domain later

### Pricing

Domains charged at-cost — exact NameSilo wholesale price, no markup. Examples (real prices):
- .com $17.29/yr
- .co $11.99/yr (renews $33.99)
- .org $10.79/yr
- .biz $2.19/yr

## 4. PAYMENTS

### How it works

- Single Stripe payment link sent in WhatsApp
- Customer taps link, pays with card
- No account creation, no sign-up
- Email receipt + WhatsApp confirmation
- Beautiful "thank-you" page with confetti after payment

### What customers pay for

- **Combined link** for website + new domain (e.g. $216.29 = $199 + $17.29 .com)
- **Website only** if they're skipping the domain ($199)
- **Website only** if they have their own domain ($199)
- **Renewal link** later (planned) — customer pays exact NameSilo renewal price each year

### Idempotent links

- Banner on the live preview site has the same Stripe link as the chat message
- Once paid, the banner disappears entirely (no risk of accidental second payment)

### Refund handling

- If customer is refunded, the site goes back to "preview" mode (banner reappears, contact forms lock)
- Stripe webhook handles this automatically

### 22-Hour Discount Window

- If customer doesn't pay within 22 hours, system automatically:
  - Applies 20% discount to the website portion (domain stays at full cost)
  - Sends a new Stripe link with the discounted total
  - Updates the preview banner to show strikethrough old price + new price + "20% OFF" badge
  - Sends WhatsApp message with the new offer
- 1 hour later (at 23h), if still unpaid, the preview site is auto-deleted from Netlify
- One-time discount per preview, no double-discounting

### No payment plans

- Pixie does NOT split website payments — full $199 upfront, period
- Discount is the only concession
- For larger services (SMM retainers, custom software), milestone splits are available

## 11. CONVERSATION MAGIC

Things that make Pixie feel less like a bot:

### Guided Dynamic flow

- No menus, no buttons, no "select an option"
- Pixie always asks the next specific question in a checklist
- One question at a time, fixed order, but skips if user volunteers info early
- User feels like they're chatting; Pixie is silently working through requirements

### Sales personality auto-detection

- Pixie reads how the customer talks and matches their style:
  - COOL: casual, emojis, low-pressure
  - PROFESSIONAL: formal, reserved
  - UNSURE: gentle, reassuring
  - NEGOTIATOR: price-conscious, direct

### Smart features

- Multi-field extraction — one message can fill 5 questions ("Glow Studio salon in Austin, haircuts and color, 512-555-0199")
- Natural language reply matching — "yeah lets do new one" = new domain, "im good without one" = skip, "use mybiz.io" = own domain
- Intent classifier prevents menu-resets mid-flow
- Entity accumulator remembers data across rapid corrections
- Message buffering merges quick consecutive messages into one turn
- Undo stack — "wait go back" works
- Rolling conversation summary (long chats stay in context)

### Edge case handling

- Off-topic question → warm redirect to website flow
- Returning visitor (after 30+ min silence) → recap message to remind context
- Multi-service intent ("I want website AND a logo") → handled in queue
- Audio messages → auto-transcribed (Whisper)
- Document/image uploads → parsed, photos stored, used in site
- Location sharing → reverse-geocoded to city
- Abuse / spam → human takeover triggered, admin notified
